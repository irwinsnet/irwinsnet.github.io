1. Defining the Kalman Filter¶

First I'll define the matrices that comprise the Kalman filter. For easy reference, here are the Kalman filter equations:

$$ \tag{1} \overline{\vec \mu}_t = \mathbf{A}_t \vec \mu_{t-1} + \mathbf{B}_t \vec u_t$$

$$ \tag{2} \overline{\mathbf{\Sigma}}_t = \mathbf A_t \mathbf \Sigma_{t-1} \mathbf A_t^T + \mathbf R $$

$$ \tag{3} \mathbf{K_t} = \mathbf{\overline{\Sigma}}_t \mathbf{{C_t}^T} (\mathbf{C_t} \mathbf{\overline{\Sigma}}_t \mathbf{{C_t}^T} + \mathbf{Q_t})^{-1} $$

$$ \tag{4} \vec \mu_t = \overline{\vec \mu}_t + \mathbf{K}_t (\vec z_t - \mathbf{C}_t \overline{\vec \mu}_t) $$

$$ \tag{6} \mathbf{\Sigma}_t = (\mathbf{I} - \mathbf{K}_t \mathbf{C}) \mathbf{\overline\Sigma}_t $$

Functions for Calculating Beliefs (Means and Covariance Matrices)¶

In [2]:
# Equation (1)
def mean_prediction(mu_1):
    return A * mu_1


# Equation (2)
def covariance_prediction(Sigma_1):
    return A * Sigma_1 * A.T + R


# Equation (3)
def kalman_gain(Sigma_pred):
    C = sympy.Matrix([[1, 0]])
    return Sigma_pred * C.T * (C * Sigma_pred * C.T + Q).inv()


# Equation (4)
def mean_post(mu_pred, z, K):
    z_matrix =sympy.Matrix([[z]])
    return mu_pred + K * (z_matrix - C * mu_pred)


# Equation (5)
def covariance_post(Sigma_pred, K):
    I = sympy.Matrix([[1, 0], [0, 1]])
    return (I - K * C) * Sigma_pred


# Function that combines equations (1) and (2)
def prediction(belief):
    return {"mu": mean_prediction(belief["mu"]),
            "Sigma": covariance_prediction(belief["Sigma"])}


# Function that combines equations (3) - (5)
def measurement(belief, z):
    K = kalman_gain(belief["Sigma"])
    return {"mu": mean_post(belief["mu"], z, K),
            "Sigma": covariance_post(belief["Sigma"], K)}

# Combine everything
def next_belief(belief, z):
    pred = prediction(belief)
    return measurement(pred, z)

def get_eigen_data(matrix):
    """Extracts eigenvalues and eigenvectors from covariance matrix.
    """
    eigenvects = matrix.eigenvects()
    if eigenvects[0][0] > eigenvects[1][0]:
        idx_max = 0
        idx_min = 1
    else:
        idx_max = 1
        idx_min = 0
        
    return {"eigvec_major": list(eigenvects[idx_max][2][0]),
            "eigvec_minor": list(eigenvects[idx_min][2][0]),
            "eigval_major": eigenvects[idx_max][0],
            "eigval_minor": eigenvects[idx_min][0],
            "theta_major": math.atan(eigenvects[idx_max][2][0][1]/
                               eigenvects[idx_max][2][0][0])*180/math.pi,
            "theta_minor": math.atan(eigenvects[idx_min][2][0][1]/
                               eigenvects[idx_min][2][0][0])*180/math.pi,
           }

Function for Plotting Beliefs¶

def plot_ellipses(beliefs, colors, title, scale = 10):
    """Plots a series of uncertainty ellipses.
    
    Args:
        data: [(mu0, cov0), (mu1, cov1), ...]
    """
    plt.close()
    
    # Setup plot
    fig = plt.figure()
    ax = fig.add_subplot(111)
  
    # Set plot size
    ax.set_xlim(-scale, scale)
    ax.set_ylim(-scale, scale)
    ax.set_aspect('equal', adjustable='datalim')
    
    # Plot ellipses
    for belief in enumerate(beliefs):
        col_idx = belief[0]
        mu = belief[1]["mu"]
        eig = get_eigen_data(belief[1]["Sigma"])
        ell = mpatches.Ellipse((mu[0], mu[1]), 
                               math.sqrt(eig["eigval_major"]*5.992)*2,
                               math.sqrt(eig["eigval_minor"]*5.992)*2,
                               eig["theta_major"], fill=False,
                               edgecolor=colors[col_idx])
        ax.add_patch(ell)
        
    # Plot ellipse centers
    x = [belief["mu"][0] for belief in beliefs]
    y = [belief["mu"][1] for belief in beliefs]
    plt.plot(x, y, ".k-")
        
    # Add title and stuff
    plt.grid()
    plt.title(title)
    plt.xlabel("Position")
    plt.ylabel("Velocity")
    plt.show()

3. Effect of Multiple Measurements on Precision¶

In Article #3 we calculated the belief at time 5 with only a single measurement at time 5. What happens if we have measurements at earlier times? Does that decrease the size of our error ellipse?

First, I'll calculate the error ellipse at time $t = 5$ with measurements at time 2 ($z = 1$) and time 5 ($z = 5$) and compare that to the error ellipse that occurs if there is only the one measurement at $t = 5$.

Effect of Measurement at Time 2¶

A plot of the 95% error ellipses for the prediction (before measurement, red) and measurement update(blue).

In [9]:
plot_ellipses([pred_2, bel_2], ["r", "b"], "Beliefs at Time t = 2", 4)

Prior to the measurement, the belief is centered at (0, 0). The effect of the measurement is to shift both $x$ and $\dot x$ in the positive direction, and to slightly shrink the covariance matrix. Even though our measurement had the robot at $x = 1$, the Kalman algorithm believes the robot is at $x = 0.2$. We can see that due to the size of the measurement uncertainty, the algorithm assigns a higher weight to the prediction.

But what if the measurement uncertainty were smaller? Let's see what would happen if we shrink the measurement covariance from 10 down to 1.

Interesting! The Kalman filter has assigned greater weight to the measurement. This results in a greater positive shift in velocity and position and a smaller error ellipse.

There are a couple interesting things going on here:

  1. The black ellipse represents the post-measurement belief at time $t = 2$. It's centered at about (0.2, 0.16) representing the measurement ($x = 1$) taken at time 2. The red ellipse represents the pre-measurement prediction at time $t = 5$. The horizontal black line tells us that the Kalman filter has shifted expected position to the right, event without a measurement. This shows that the filter is able to account for the believed velocity at time $t = 2$ ($\dot x = 0.16$) and incorporate it into subsequent beliefs.
  2. The blue ellipse represents the measurement update at time $t = 5$. The measurement has resulted in significantly shrinking the uncertainty. Also, the believed position is now at about 4.05, not far from the measurement of 5. In this instance the filter weighted the measurement more than the prediction (because the prediction had much greater uncertainty than the measurement).

Now let's see what the belief at time $t = 5$ would be with no measurement at time $t = 2$.

Hmmm. Not much difference. This must be because the measurement covariance is so high, that there was little difference between the prediction and update at time 2. Let's see what happens if we recalculate with measurement covariance = 1.

Now that's more like it. The error ellipse is much smaller when there is an earlier measurement at time 2 (green). Also note that the filter has assigned a larger position and uncertainty.

When the covariance of the measurement is set to 1, that results in significantly shrinking the belief uncertainty at time $t = 2$. That results in less uncertainty at later times than what would have occurred if the measurement at time $t = 2$ had not occurred.
